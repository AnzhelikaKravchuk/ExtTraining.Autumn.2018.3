Так как PasswordCheckerService был "жестко" связан с Sqlrepository, не было возможности работать с репозиториями других типов. Для разрешения данной проблемы был создан интерфейс IRepository,  и в PasswordCheckerService  был создан параметризованный конструктор, принимающий интерфейсную переменную на репозиторий клиента.
